# 种子数据 `data/`

这是**菜品库 / 食材库 / 时令表 / 平台搜索词**的起点数据。
当前已经扩展成可用的 MVP 种子库：302 道深圳家庭常见菜、165 个食材、75 条营养标签、165 条价格基线。菜谱步骤为自有模板生成，不复制商业网站、用户菜谱或旧菜谱正文。

| 文件 | 用途 | 目标规模（MVP） | 最终规模 |
|------|------|----------------|----------|
| `dishes.seed.json` | 菜品库，当前为原创结构化深圳家常菜 | 300 道 | 1000-2000 道 |
| `ingredients.seed.json` | 食材库 + 别名 + 平台搜索词 | 200 个 | 500+ |
| `season-shenzhen.json` | 深圳/华南时令表 | 12 月全覆盖 | 不变 |
| `platform-keywords.json` | 不同平台的搜索关键字映射 | 覆盖常见食材 | 随食材库扩展 |
| `nutrition.seed.json` | 营养标签映射，后续对接中国食物成分数据中心 | 200 个 | 500+ |
| `price-baseline.json` | 本地价格基线，后续对接农业农村部菜篮子价格 | 100 个 | 300+ |
| `data-sources.json` | 数据来源与版权策略说明 | 不变 | 不变 |

## 当前构建方式

当前数据由 `scripts/build-food-data.cjs` 生成：

```bash
node scripts/build-food-data.cjs
```

策略：

- 不复制下厨房、豆果、美食杰等商业网站菜谱正文。
- 《中国菜谱（广东）》等旧菜谱只作为菜名、粤菜技法和目录参考；入库做法使用本项目自有家常步骤模板重写。
- Cook1Cook 等网站分类页只作为菜名目录候选；不抓取详情页步骤，不复刻用户发布的菜谱正文。
- 先维护深圳家庭常见菜的结构化事实：菜名、食材、时间、难度、适合人群、做法模板。
- 菜谱步骤用自有模板生成，避免版权风险。
- 价格基线先用本地占位；牛羊肉、贵价水产等高价食材参考公开价格监测做人工基线，后续映射农业农村部主要菜篮子价格日动态。
- 营养标签先用本地占位，后续映射中国食物成分数据中心。
